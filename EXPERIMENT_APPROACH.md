# Experiment Approach

This project studies how large monocular depth models can be used to improve a smaller single-view depth model for a target domain. The broader motivation is that the largest foundation models already perform very well zero-shot, but compact models are often the useful deployment target when inference has to run on limited hardware, edge devices, or latency-constrained systems. Our experiments therefore ask whether a compact student can be specialized to the benchmark domain without damaging the pretrained geometric priors it already learned during large-scale distillation.

The experiments are structured around four baselines.

## Baseline 1: Zero-Shot Student Evaluation

Baseline 1 evaluates the pretrained DA3MONO-LARGE student directly on the test set without task-specific training. This establishes the reference point for the compact model before any downstream adaptation.

The important role of this baseline is to measure the value of the pretrained prior itself. Since DA3MONO-LARGE has already been trained with large-scale teacher-student supervision, any later fine-tuning method must improve on this strong prior rather than simply show that the model can fit the training set.

## Baseline 2: Ground-Truth Fine-Tuning

Baseline 2 fine-tunes the student on the provided ground-truth depth labels. The conservative version updates only the prediction head while keeping the backbone frozen, with variants that expose either the full head or a smaller surface portion of the head.

This baseline tests the standard downstream adaptation strategy: use the task labels directly and validate against a held-out subset of the same labels. In our setting, this proved fragile. Even when the optimization was made conservative and the loss was aligned with the benchmark-style per-image scale-invariant log objective, ground-truth fine-tuning often degraded the zero-shot student. The interpretation is that the provided labels may be noisy, inconsistent in scale, or less geometrically coherent than the pretrained prior already encoded in the model.

## Baseline 3: Teacher-Guided Pseudo-Label Adaptation

Baseline 3 replaces the direct ground-truth training target with pseudo-labels generated by larger DA3 teacher models. We generate teacher predictions for the training images, then fine-tune DA3MONO-LARGE on those pseudo-depth maps. Ground-truth labels are still used for validation and checkpoint selection, but not as the training target in the main pseudo-label adaptation runs.

This baseline is motivated by the same teacher-student paradigm that made the Depth Anything family effective during pretraining. Instead of treating pseudo-labeling as only an upstream pretraining strategy, we reuse it as a post-pretraining domain adaptation mechanism. The larger teacher provides a smoother and more compatible supervision signal for the compact student than the raw benchmark labels.

The final Baseline 3 setup also adds distribution-aware validation selection. A random validation split can be misleading when the benchmark test set is concentrated around particular scene types while the training set is broader. To make checkpoint selection better reflect the target distribution, we extract pooled DA3/DINO backbone features for both train and test images and rank training images by similarity to the test set. We then build validation manifests from the top 10 percent most test-similar training images.

We implemented two retrieval variants:

- **Cosine feature retrieval:** compare the pooled backbone feature vectors directly with cosine similarity.
- **PCA retrieval:** reduce the pooled features with PCA before nearest-neighbor retrieval, with a separate 2D PCA plot used only for visualization/reporting.

Both methods produce CSV manifests that Baseline 3 can consume through the manifest-validation option. The training set can still include all pseudo-labeled training images, while validation is performed on the selected ground-truth subset. This separates the supervision source used for optimization from the target-like subset used for checkpoint selection.

## Baseline 4: Targeted Adaptation and Novel-View Augmentation

Baseline 4 starts from the observation that the Baseline 3 validation selection also identifies the training images most visually similar to the test set. We therefore use the top cosine-ranked images as a targeted subset for final adaptation experiments.

The first Baseline 4 ablation fine-tunes from the best Baseline 3 checkpoint on only the top 250 test-similar training images, supervised by DA3-GIANT pseudo-labels. This isolates the effect of short, target-distribution adaptation without introducing synthetic views.

The second Baseline 4 ablation tests novel-view synthesis as geometric data augmentation. For each of the same top 250 images, we keep the original image and teacher pseudo-depth, then render four nearby synthetic viewpoints: left, right, up, and down. The novel-view RGB images and corresponding depth targets are produced from the teacher-guided Gaussian-splat representation, with a confidence mask retaining the most reliable projected pixels.

This augmentation did not work well enough to improve the final model. Visual inspection showed that the rendered images were often blurry and that the reprojected depth maps failed to preserve fine surface-depth structure. The generated targets therefore became a noisier learning signal than the original teacher pseudo-labels. We still treat the experiment as useful evidence: synthetic view augmentation remains conceptually attractive, but in this setting single-image novel-view synthesis did not provide depth targets with sufficient geometric fidelity.

## Overall Interpretation

The intended story is not to beat the zero-shot performance of the largest teacher models. Instead, the main question is whether a compact student can be improved through careful supervision from larger models, especially when direct labels are noisy and deployment constraints make smaller models more useful.

The resulting progression is:

1. zero-shot compact student as a strong pretrained prior;
2. direct ground-truth fine-tuning, which can damage that prior;
3. teacher pseudo-label adaptation, which gives a more stable supervision signal and benefits from target-aware validation clustering;
4. targeted adaptation and novel-view augmentation, where the targeted subset is useful but low-fidelity generated views are not reliable enough as depth supervision.
